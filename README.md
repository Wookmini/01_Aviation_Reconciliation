# 항공권 정산 자동화 시스템 (Aviation Reconciliation Automation)

본 프로젝트는 기업용 결제 시스템(e-DAHS)과 여행사(세중) 리스트 간의 막대한 데이터를 대조하는 과정에서 발생하는 비효율을 해결하기 위한 엔지니어링 솔루션입니다.

## 1. Background: 비즈니스 컨텍스트
전사적 자원 관리의 일환으로 발생하는 항공권 정산 작업은 전통적으로 극심한 수작업을 요구해왔습니다. 특히 **e-DAHS(사내 결제 시스템)**의 데이터와 **세중(여행사)**에서 제공하는 리스트 간의 항목별 교차 검증은 단순 반복을 넘어, 미세한 금액 차이로 인한 회계 불일치 리스크를 상시 내포하고 있었습니다. 본 시스템은 이러한 페인 포인트를 제거하고, 정산의 무결성을 보장하기 위해 설계되었습니다.

## 2. The Logic: 핵심 매칭 전략
본 시스템의 핵심은 단순한 텍스트 비교를 넘어선 **'승인번호(Approval Number)' 기반의 다차원 매칭 전략**에 있습니다.

- **승인번호 기반 정밀 타격**: 데이터 간의 유일한 식별자인 승인번호를 핵심 키로 사용하여 오매칭 확률을 제로화합니다.
- **v9의 동적 행 처리(Dynamic Rows)**: 버전 9에서는 고정된 양식을 탈피하여, 여행사 리스트의 불규칙한 행 구조와 변동적인 데이터 배치를 실시간으로 분석하고 파싱하는 유연성을 갖추었습니다.
- **Net Amt vs Total Amt 검증**: 단순 합계 비교가 아닌, 공급가액(Net)과 총액(Total)의 상관관계를 분리하여 검증합니다. 이는 유류할증료나 세금 항목에서 발생하는 미세한 오차를 잡아내어 금융 사고를 사전에 방지하는 핵심 로직입니다.

## 3. Visual Feedback: 비즈니스 인사이트
분석 결과는 실무자가 직관적으로 판단할 수 있도록 시각적 가이드라인을 제공합니다.

- **Red (Error)**: 승인번호는 일치하나 금액이 불일치하는 경우입니다. 이는 결제 프로세스 상의 오류나 환율 적용의 미세한 차이를 의미하며, 즉각적인 재검토가 필요합니다.
- **Yellow (Missing)**: 한쪽 리스트에는 존재하나 대응되는 항목이 없는 경우입니다. 누락된 영수증이나 미처리된 결제 내역을 식별하여 정산 누락을 방지합니다.

## 프로젝트 구조
- `run_reconciliation.py`: 메인 정산 및 대조 로직 스크립트 (최종 버전)
- `analyze_excel.py`: 엑셀 파일 구조 분석 및 전처리 유틸리티
- `check_rows.py`: 데이터 행 수 및 기본 무결성 검사
- `excel_workspace/`: 정산에 필요한 데이터 파일 보관

## 시작하기
```bash
pip install pandas openpyxl
python run_reconciliation.py
```
